# 🚀 Irtaqi Moodle Auto Sync Worker

## ✨ المزايا

✅ **مزامنة تلقائية 24/7** - يعمل كل ساعة بدون توقف  
✅ **لا يحتاج فتح المودل** - يعمل في الخلفية  
✅ **مجاني تماماً** - 10,000 طلب/يوم مجاناً  
✅ **سريع جداً** - Cloudflare Edge Network  
✅ **آمن** - بيانات الدخول مشفرة  

---

## 📦 التثبيت والنشر

### المتطلبات:

1. **حساب Cloudflare** (مجاني)
2. **Node.js** مثبت على جهازك
3. **بيانات دخول المودل** (اسم المستخدم وكلمة المرور)

---

### الخطوة 1: تثبيت Wrangler CLI

افتح PowerShell وشغّل:

```bash
npm install -g wrangler
```

---

### الخطوة 2: تسجيل الدخول لـ Cloudflare

```bash
wrangler login
```

سيفتح متصفح لتسجيل الدخول.

---

### الخطوة 3: إضافة المتغيرات السرية

في مجلد `cloudflare-worker`، شغّل:

```bash
# إضافة اسم المستخدم في المودل
wrangler secret put MOODLE_USERNAME

# إضافة كلمة المرور
wrangler secret put MOODLE_PASSWORD

# إضافة Supabase Key
wrangler secret put SUPABASE_KEY
```

عند كل أمر، سيطلب منك إدخال القيمة.

**القيم المطلوبة:**

- `MOODLE_USERNAME`: اسم المستخدم في المودل
- `MOODLE_PASSWORD`: كلمة المرور
- `SUPABASE_KEY`: المفتاح من Supabase

## 🎯 كيف يعمل

### المزامنة التلقائية:

- ⏰ **كل ساعة** يعمل Worker تلقائياً
- 🔍 **يفحص المودل** للملفات الجديدة
- 📥 **يحمّل الملفات** من المودل
- ☁️ **يرفعها** إلى Supabase
- 💾 **يحفظ المعلومات** في قاعدة البيانات

### المزامنة اليدوية:

يمكنك تشغيل المزامنة يدوياً بزيارة:

```
https://irtaqi-moodle-sync.YOUR-SUBDOMAIN.workers.dev/sync
```

(استبدل `YOUR-SUBDOMAIN` بالرابط الذي يعطيك إياه Cloudflare)

## ⚙️ التخصيص

### تغيير وقت المزامنة:

في `wrangler.toml`، عدّل السطر:

```toml
crons = ["0 * * * *"]
```

**أمثلة:**

- كل 30 دقيقة: `"*/30 * * * *"`
- كل 6 ساعات: `"0 */6 * * *"`
- مرة يومياً (الساعة 2 صباحاً): `"0 2 * * *"`

## 🛠️ استكشاف الأخطاء

### المشكلة: Worker لا يعمل

**الحل:**
```bash
wrangler tail
```
شاهد الأخطاء في السجلات.

### المشكلة: فشل تسجيل الدخول للمودل

**الحل:**
- تأكد من صحة اسم المستخدم وكلمة المرور
- جرب تسجيل الدخول يدوياً في المودل

### المشكلة: لا يتم رفع الملفات

**الحل:**
- تأكد من صحة Supabase Key
- تحقق من صلاحيات Storage في Supabase
